## 第一部分：执行摘要与核心定位

### 1.1 游戏身份定位

| 维度 | 内容 |
|------|------|
| **品类融合** | RTS基地建造 + 双摇杆射击(Twin-Stick Shooter) + 塔防 + 生存 |
| **开发者类比** | "Diablo meets Anno" / "Factorio with combat" |
| **核心设定** | 玩家同时扮演战斗者(驾驶机甲Mr.Riggs)和建设者(3D打印无人机建造基地) |
| **平台** | PC / PS5 / Xbox Series X/S |
| **玩家数** | 1-4人(2.0更新后支持联机) |
| **单局时长** | 战役: 15-30小时 / 生存: 60分钟基准(可自定义) |
| **商业模式** | 买断制 + 3个DLC扩展包 |

### 1.2 核心设计哲学 (四大支柱)

| 支柱 | 描述 | 实现机制 |
|------|------|----------|
| **双重身份张力** | 玩家必须在个人战斗和基地建设之间不断切换，两者缺一不可 | 机甲可离开基地战斗，但基地需要持续维护；离开太久基地会被摧毁 |
| **扩张即挑衅** | 每一次资源采集和基地扩张都会增加星球敌意，触发更强攻击 | 生物群落破坏度系统 + 波次强度与基地规模挂钩 |
| **有限资源焦虑** | 矿脉会枯竭，迫使玩家不断向危险区域扩张 | 所有矿脉有固定储量，采完即消失 |
| **电力即生命线** | 电力是全局共享资源，断电=防御瘫痪=基地毁灭 | 全局电网系统，发电<用电时所有建筑停摆 |

### 1.3 成功指标参考

- **Steam评价**: 91% Positive (13,901 reviews)
- **Metacritic评论家**: 8.3/10
- **Metacritic用户**: 8.4/10
- **平均游戏时长**: 10.8+ hours (核心玩家)
- **2.0更新后崩溃率**: 0.67% per session (从8%下降)

## 第二部分：核心玩法循环

### 2.1 循环详细拆解

```
探索 → 选址 → 采集 → 建造 → 研究 → 防御 → 波次抵御 → 扩张/升级 → 循环
```

| 步骤 | 名称 | 持续时间 | 核心行动 | 风险 | 回报 |
|------|------|----------|----------|------|------|
| 1 | **探索** | 持续 | 驾驶机甲探索地图、扫描生物、发现资源点 | 遭遇敌对生物，远离基地无法回防 | 新资源点、DNA样本、地图信息 |
| 2 | **选址** | 一次性 | 放置HQ/前哨站，选择靠近双矿位置 | 位置不佳=资源枯竭快=频繁搬迁 | 建立永久基地，解锁建造研究 |
| 3 | **采集** | 持续 | 建造矿场、建立电力、铺设管道 | 矿脉有限，过度开采=资源枯竭 | 持续资源流入，支撑建造研究 |
| 4 | **建造** | 持续 | 防御塔、城墙、生产建筑、储能设施 | 建造消耗资源，错误布局=防御漏洞 | 基地功能完善，防御能力增强 |
| 5 | **研究** | 队列式 | 选择研究项目、支付资源、等待完成 | 研究耗时，期间无法解锁新科技 | 新建筑、新武器、新能力、升级 |
| 6 | **防御** | 波次触发 | 操作机甲战斗、依赖自动炮塔、修复建筑 | 防御失败=建筑被毁=资源损失 | 生存下来，保留建设成果 |
| 7 | **扩张** | 周期性 | 建立分基地、建造Rift Portal、跨地图传送 | 多线作战，资源分散 | 新资源、新科技、战役推进 |

### 2.2 关键张力设计点

| 张力 | 描述 | 机制实现 |
|------|------|----------|
| **探索 vs 防守** | 想探索远处获取稀有资源，但离开基地时波次可能来袭 | Rift Portal快速传送、基地警报系统、自动防御塔 |
| **扩张 vs 安全** | 建造更多建筑=更大目标=更强攻击波次 | 波次强度与HQ等级/建筑数量挂钩 |
| **短期 vs 长期** | 立即建造防御 vs 投资研究解锁更强科技 | 研究树分层，关键科技需要前置条件 |
| **个人 vs 基地** | 机甲亲自战斗 vs 建造更多防御塔自动防御 | 机甲有最强DPS但无法同时出现在所有位置 |

### 3.2 采矿机制详解

**核心规则**: 矿脉由多个 **square** 组成，每个 square 有固定储量。矿场只能采集 footprint 范围内的 square。

#### 矿场覆盖规则

- **3x3矿场** = 覆盖9个square
- 每个square独立计算储量
- **示例**: 覆盖9个square的矿场，每个square 100单位 = 总储量900，产出+1/s，可持续15分钟

#### 产出与储量关系

- **产出速度**: 固定(+1/s Lv1)，与覆盖square数量无关
- **储量决定**: 持续时间
- **策略选择**:
  - 覆盖更多square = 更持久，但占地更大
  - 覆盖少square = 快速枯竭但可密集布置

#### 矿脉枯竭

- square采完后矿场自动停止
- 建筑保留但无产出
- **设计意图**: 迫使玩家寻找新矿脉，推动地图探索

### 3.4 电力系统 (核心瓶颈设计)

| 发电机 | 产出 | 条件 | 解锁 |
|--------|------|------|------|
| **Wind Turbine** | 12/s | 连续运行，天气事件时停转 | 初始 |

## 第四部分：科技树系统

### 4.1 三大研究分支

| 分支 | 解锁建筑 | 内容 | 依赖关系 |
|------|----------|------|----------|
| **Base and Buildings** | Communication Hub | 防御结构、能源、资源、弹药、HQ升级 | 部分锁定在HQ等级后 |
| **Weapons and Equipment** | Communication Hub | 机甲武器、升级、消耗品、移动技能 | 部分锁定在Armory等级后，Armory又依赖HQ |
| **Alien Research** | Laboratory + 清洁水源 | 环境保护、特殊资源、实验室升级、特殊技术 | 需要DNA Fragments(击杀/扫描生物) |

### 4.2 研究机制差异

#### 战役模式

- **要求**: 建造Communication Hub下载蓝图
- **成本**: 时间 + 资源 (2.0更新后)
- **队列**: 可排队研究
- **解锁条件**: 部分需要完成剧情任务

#### 生存模式

- **独立研究树**: 初始解锁部分基础科技
- **Alien Research**: 可直接从Communication Hub访问，无需Laboratory
- **移除战役目标**: 所有研究只需前置科技完成

### 4.3 研究资源

| 资源 | 来源 | 用途 |
|------|------|------|
| **Download** | Communication Hub产出 | Base/Weapons科技 |
| **Analysis** | Alien Lab产出 | Alien科技 |

### 4.4 DLC扩展

每个DLC新增 **40+ 研究项**，扩展科技树深度。

| **Wall** | 阻挡敌人路径 | 可升级: 普通→能量墙→水晶墙→藤蔓墙 |
| **Gate** | 可开关的通道 | 与墙配套 |
| **AI Hub** | 自动修复范围内建筑 | 无人机修复 |
| **Shield Generator** | 生成护盾保护建筑 | 吸收伤害 |
| **Repair Facility** | 修复受损建筑 | 无人机修复，详见3.3 |
| **Ammunition Tower** | 生产并输送弹药 | 防御塔需要弹药补给 |

### 6.7 弹药物流系统

**核心机制**: 防御塔需要弹药补给，由弹药塔生产并输送。

**设计意图**: 弹药塔必须靠近防御塔布置，或建立物流网络，增加布局复杂度。

## 第七部分：波次攻击系统

### 7.1 波次强度公式

```
n = 2 + 9 * e^(0.3r)
```

- **n**: 敌人数量
- **r**: 波次数

| 波次 | 敌人数量 |
|------|----------|
| 1 | ~14 |
| 5 | ~50+ |
| 8 | ~100+ |
| 15 | ~300+ |

### 7.2 波次时间参数

| 难度 | 首波时间 | 间隔 | 首次强度增长 | 后续强度增长 |
|------|----------|------|-------------|-------------|
| **Easy** | 5分钟 | 8分钟 | 5分钟 | 12分钟 |
| **Normal** | 5分钟 | 7分钟 | 3:20 | 10分钟 |
| **Hard** | 5分钟 | 7分钟 | 3:20 | 10分钟 |
| **Brutal** | 5分钟 | 7分钟 | 3:20 | 8:20 |

### 7.3 难度缩放公式

```
Difficulty = creatures_base_difficulty + creatures_distance_per_difficulty / distance_from_player_initial_spawn
```

| 变量 | 说明 | Easy | Normal | Hard | Brutal |
|------|------|------|--------|------|--------|
| creatures_base_difficulty | 基础难度 | 0 | 0 | 0.5 | 1.0 |
| creatures_distance_per_difficulty | 距离系数 | 512 | 256 | 256 | 256 |
| creatures_max_difficulty | 最大难度 | 4 | none | none | none |

**设计意图**: 离出生点越远敌人越强，鼓励先清理近处。

### 7.4 敌人生成机制

| 机制 | 描述 |
|------|------|
| **Random Scatter** | 敌人在地图边界随机生成，向基地推进 |
| **Swarm Cluster** | 后期出现大规模集群，需要协作应对 |
| **Boss Waves** | 特定波次减少普通敌人数量，聚焦Boss战 |
| **Ambient Spawns** | 非波次期间也有ambient生物游荡 |

### 7.5 波次难度参数表

| 参数 | Easy | Normal | Hard | Brutal |
|------|------|--------|------|--------|
| 敌人伤害倍率 | 0.33 | 1.0 | 1.0 | 1.0 |
| 资源储量倍率 | 2 | 1 | 1 | 1 |
| 生存时间 | 60分钟 | 60分钟 | 90分钟 | 90分钟 |

### 12.1 核心循环实现优先级

### 12.3 可复用设计模式

#### 模式1: 扩张-惩罚循环

```
玩家扩张 → 增加威胁 → 更强攻击 → 需要更强防御 → 需要更多资源 → 继续扩张
```

**实现**: 波次强度与HQ等级/建筑数量挂钩

#### 模式2: 电力瓶颈

**设计**: 电力永远不够，迫使玩家做选择
**实现**: 全局电网，发电<用电时全局停摆

#### 模式3: 资源枯竭

**设计**: 矿脉有限，推动持续探索
**实现**: 每个square固定储量，采完停止

#### 模式4: 元素相克

**设计**: 敌人有抗性，必须混合防御
**实现**: 7种伤害类型，敌人随机免疫1-2种

#### 模式5: 双重身份

**设计**: 玩家既是战士又是建设者
**实现**: 机甲离开基地时基地脆弱，必须建造自动防御
