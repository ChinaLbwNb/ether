# 《Ether》角色与建筑资产规划

## 1. 资产原则

本项目参考《The Riftbreaker》的系统结构，但资产必须原创或使用明确可商用授权的素材，不能直接使用原游戏角色、建筑、图标、音效、UI 或截图资源。

资产开发采用三阶段策略：

1. 原型期：用简单形状、纯色图块、低成本占位图完成玩法验证。
2. 风格期：确定机甲、基地、敌人、资源点的统一美术方向。
3. 产品期：替换为完整动画、特效、音效和 UI 图标。

## 7. 推荐落地方式

### 7.1 v0.1 原型

先用 Godot 原生节点完成：

- 基地：`Sprite2D` 或 `Polygon2D` 占位。
- 防御塔：底座圆形 + 炮管矩形。
- 敌人：红色圆形或简易 Sprite。
- 资源点：发光蓝色矿石占位。
- 子弹：小圆点或短线。

这样可以最快完成玩法闭环，不会因为资产等待而卡住。

### 7.2 v0.2 风格确定

确定统一方向：

- 科幻机甲。
- 异星工业基地。
- 能量矿与生态污染。
- 高对比阵营色：玩家蓝白/青色，敌人红紫/橙色。

### 7.3 v0.3 资产替换

逐步替换：

1. 玩家机甲完整动画。
2. 基地核心。
3. 哨兵塔。
4. 第一种敌人。
5. 资源点。
6. UI 图标。
7. 子弹和命中特效。

## 9. 近期建议

近期不要先追求完整精美资产。最合适的顺序是：

1. 用占位资产完成 `v0.1 基地防守原型`。
2. 确认防守流程好玩。
3. 再制作基地核心、哨兵塔、小怪和资源点的第一版正式资产。

这样可以避免美术方向提前锁死，也能让资产服务玩法，而不是反过来拖住玩法。

## 11. 玩家机甲完整动作需求

### 11.1 基础移动动作

建议最终采用 8 方向或“身体 4 方向 + 武器独立瞄准”的方案。早期可以先保留正面/背面，后期再补齐。

| 动作 | v0.1 | 完整版 | 说明 |
| --- | --- | --- | --- |
| 待机 | 正面、背面 | 8 方向或 4 方向 | 当前已有正面/背面 |
| 行走/奔跑 | 正面、背面 | 8 方向或 4 方向 | 后期需要左右方向 |
| 转向 | 不需要 | 需要 | 让角色朝向更顺滑 |
| 冲刺 | 不需要 | 8 方向 | 机甲机动核心 |
| 停步刹车 | 不需要 | 可选 | 增强重量感 |
| 受击 | 简单闪烁 | 方向无关/方向相关 | 反馈玩家受伤 |
| 击退 | 不需要 | 需要 | 对应强敌攻击 |
| 死亡 | 不需要 | 需要 | 失败或复活流程 |
| 复活 | 不需要 | 需要 | 基地复活仓/传送复活 |

### 11.2 交互动作

| 动作 | 用途 | 资产需求 |
| --- | --- | --- |

| --- | --- | --- |
| 小型近战怪 | 前期基础压力 | 移动、攻击、受击、死亡 |
| 群体虫群 | 数量压迫 | 小体型、快速移动、群体死亡 |
| 重甲怪 | 吃火力 | 厚甲、慢走、重击、破甲反馈 |
| 远程怪 | 打塔/打墙 | 蓄力、发射物、远程命中 |
| 破墙怪 | 针对城墙 | 撞击/啃咬墙体、冲锋 |
| 自爆怪 | 强制优先击杀 | 闪烁警告、自爆特效 |
| 飞行怪 | 绕过城墙 | 飞行、俯冲、空中死亡 |
| 掘地怪 | 突袭后方 | 入地、出土、土尘 |
| 护盾怪 | 保护小怪 | 护盾光环、护盾破碎 |
| 治疗怪 | 延长战斗 | 治疗光束、治疗范围 |
| 精英怪 | 波次小 Boss | 强调色、特殊技能 |
| Boss | 关卡核心挑战 | 多阶段动画、技能预警、死亡演出 |
